# Movimiento de datos

1. Realiza una exportación del esquema de SCOTT usando Oracle Data Pump con las siguientes condiciones:

  • Exporta tanto la estructura de las tablas como los datos de las mismas.

  • Excluye la tabla BONUS y los departamentos con menos de dos empleados.

  • Realiza una estimación previa del tamaño necesario para el fichero de exportación.

  • Programa la operación para dentro de 2 minutos.

  • Genera un archivo de log en el directorio raíz.

Para almacenar el archivo de exportación y el archivo de registro, creamos un directorio en el sistema de archivos de Linux.

mkdir /home/oracle/dir

Accedemos a SQL*Plus para crear un directorio en la base de datos mediante la ejecución del siguiente comando:

create directory dmp_dir AS '/home/oracle/dir';

Para que el usuario SCOTT pueda leer y escribir en el directorio que creamos, es necesario asignarle los permisos correspondientes. Es crucial asegurarnos de que estos permisos sean otorgados para garantizar el correcto funcionamiento del sistema.

grant read, write on directory dmp_dir to SCOTT;

Además, es imprescindible conceder permisos al usuario SCOTT para realizar una exportación completa. Esto permitirá que SCOTT tenga los privilegios necesarios para exportar datos y garantizar el correcto funcionamiento del sistema.

grant datapump_exp_full_database to SCOTT;

Antes de exportar los datos, realizaremos una estimación aproximada del tamaño necesario para el archivo de exportación. Para ello, utilizamos el comando expdp junto con la opción ESTIMATE_ONLY=y. Este comando nos permitirá calcular el tamaño aproximado del archivo de exportación y planificar adecuadamente la cantidad de espacio de almacenamiento que necesitaremos.

expdp scott/tiger DIRECTORY=dmp_dir CONTENT=ALL LOGFILE=exportado.log SCHEMAS=SCOTT ESTIMATE_ONLY=y

A continuación, procedemos a utilizar el comando expdp para exportar el esquema SCOTT con las condiciones específicas solicitadas. Este comando nos permitirá crear un archivo de exportación con todos los datos del esquema SCOTT, siguiendo las condiciones especificadas previamente.

expdp scott/tiger DIRECTORY=dmp_dir DUMPFILE=scott.dmp CONTENT=ALL LOGFILE=scott.log SCHEMAS=SCOTT EXCLUDE=TABLE:"IN('BONUS')" QUERY=DEPT:'"where deptno in (select deptno from emp group by deptno having count(*) > 2)"' JOB_NAME=scott_exportado

Esperaremos a que se complete el trabajo de exportación. El archivo de exportación y el archivo de registro se almacenarán en el directorio dir.

2. Importa el fichero obtenido anteriormente usando Oracle Data Pump pero en un usuario distinto de la misma base de datos.

Voy a establecer el perfil del usuario objetivo en la base de datos, asignándole los permisos correspondientes.

create user importar identified by passwd;
alter user importar account unlock;
grant connect, resource to importar;
grant unlimited tablespace to importar;
alter user importar default tablespace users;
alter user importar temporary tablespace temp;
grant read, write on directory dmp_dir to importar;
grant imp_full_database to importar;

Utilizaremos el comando impdp para llevar a cabo la importación.

impdp importar/passwd DIRECTORY=dmp_dir DUMPFILE=scott.dmp LOGFILE=ScottImportado.log SCHEMAS=SCOTT REMAP_SCHEMA=SCOTT:importar

3. Realiza una exportación de la estructura de todas las tablas de la base de datos usando el comando expdp de Oracle Data Pump probando al menos cinco de las posibles opciones que ofrece dicho comando y documentándolas adecuadamente.

Oracle Data Pump cuenta con el comando expdp que permite exportar objetos y datos en una base de datos Oracle. A continuación, presentaré cinco de las opciones disponibles en expdp que se pueden emplear:

  • SCHEMAS se utiliza para indicar los esquemas que deben ser exportados.

Antes de ejecutar el comando, crearé el nuevo esquema:

grant connect, resource, unlimited tablespace to SCOTTFALSE identified by tiger;
alter user SCOTTFALSE account unlock;
alter user SCOTTFALSE default tablespace users;
alter user SCOTTFALSE temporary tablespace temp;
connect SCOTTFALSE/tiger;

create table filiales (
    id NUMBER(10) PRIMARY KEY,
    nombre VARCHAR2(50) NOT NULL,
    direccion VARCHAR2(100) NOT NULL,
    ciudad VARCHAR2(50) NOT NULL,
    pais VARCHAR2(50) NOT NULL
);

insert into filiales values (1, 'Filiales Corporation', '123 Main St', 'Ciudad de Mexico', 'Mexico');
insert into filiales values (2, 'Filiales SA de CV', '456 Calle Principal', 'Guadalajara', 'Mexico');
insert into filiales values (3, 'Filiales Chile Ltda', '789 Avenida Central', 'Santiago', 'Chile');

Puedes emplear el siguiente comando para exportar solamente los esquemas HR y SCOTT:

expdp scott/tiger DIRECTORY=dmp_dir SCHEMAS=SCOTTFALSE,SCOTT DUMPFILE=exportado1.dmp LOGFILE=exportado1.log

  • INCLUDE se emplea para definir qué tipos de objetos deben ser incluidos en el proceso de exportación. Por ejemplo, si se desea exportar únicamente las restricciones, se puede utilizar el comando siguiente:
expdp scott/tiger DIRECTORY=dmp_dir SCHEMAS=SCOTT INCLUDE=CONSTRAINT DUMPFILE=exportado2.dmp LOGFILE=exportado2.log

Vamos a mirar el contenido del fichero de log de exportación:

  • EXCLUDE se usa para indicar qué tipos de objetos deben ser excluidos del proceso de exportación. Por ejemplo, si se desea excluir la tabla "DEPT" y las restricciones almacenadas de la exportación, se puede emplear el siguiente comando:
expdp scott/tiger DIRECTORY=dmp_dir SCHEMAS=SCOTT EXCLUDE=TABLE:"IN('DEPT')",CONSTRAINT DUMPFILE=exportado3.dmp LOGFILE=exportado3.log

Vamos a mirar el contenido del fichero de log de exportación:

  • COMPRESSION se usa para determinar el grado de compresión que se aplicará a los datos que se exporten. Se pueden emplear los valores ALL (compresión máxima), METADATA_ONLY (sin compresión) y un número del 1 al 9 que indica el nivel de compresión deseado. Por ejemplo, si se desea utilizar la máxima compresión, se puede utilizar el siguiente comando:
expdp scott/tiger DIRECTORY=dmp_dir SCHEMAS=SCOTT COMPRESSION=ALL DUMPFILE=exportado4.dmp LOGFILE=exportado4.log

Podemos ver que el archivo de exportación es mucho más pequeño que otros que no han sido comprimidos.

Vamos a mirar el contenido del fichero de log de exportación:

  • ENCRYPTION_PASSWORD se usa para establecer la contraseña que se empleará para cifrar los datos que se exporten. Por ejemplo, si se desea utilizar la contraseña "sql", se puede utilizar el siguiente comando:
expdp scott/tiger DIRECTORY=dmp_dir SCHEMAS=SCOTT ENCRYPTION_PASSWORD=sql DUMPFILE=exportado5.dmp LOGFILE=exportado5.log

Voy a generar el usuario de destino en la base de datos y le asignaré los permisos adecuados.

create user importar2 identified by passwd;
alter user importar2 account unlock;
grant connect, resource to importar2;
grant unlimited tablespace to importar2;
alter user importar2 default tablespace users;
alter user importar2 temporary tablespace temp;
grant read, write on directory dmp_dir to importar2;
grant imp_full_database to importar2;

En este momento, procederemos a importar el archivo de exportación que hemos creado previamente.

impdp importar2/passwd DIRECTORY=dmp_dir DUMPFILE=exportado5.dmp LOGFILE=importado5.log

Como podemos observar, es necesario proporcionar un parámetro que contenga la contraseña para descifrar los datos. De no hacerlo, se mostrará el mismo error que hemos visto antes.

impdp importar2/passwd DIRECTORY=dmp_dir DUMPFILE=exportado5.dmp LOGFILE=importado5.log ENCRYPTION_PASSWORD=sql

Podemos verificar que el archivo de exportación ha sido importado con éxito.

Estos ejemplos que hemos visto son solo una muestra de las opciones que expdp ofrece. Es crucial revisar la documentación de Oracle para conocer todas las opciones y cómo aplicarlas según las necesidades específicas de cada caso de uso. Lo que hemos explicado aquí solo representa una fracción de las capacidades que expdp ofrece.

Otra manera de conocer las opciones disponibles en expdp es mediante la salida del comando expdp help=y. Esto nos mostrará una lista completa de las opciones disponibles junto con su descripción correspondiente.

4. Intenta realizar operaciones similares de importación y exportación con las herramientas proporcionadas con MySQL desde línea de comandos, documentando el proceso.

MySQL ofrece una herramienta llamada mysqldump que permite exportar datos desde la línea de comandos.

Exportación de datos con mysqldump

mysqldump es una herramienta útil en MySQL que permite crear copias de seguridad de bases de datos. Con mysqldump, se pueden exportar todas las tablas, vistas o rutinas almacenadas de una base de datos o simplemente una selección de ellas. Si se desea exportar completamente una base de datos, se puede emplear el siguiente comando:

mysqldump -u scott -p scott > ScottExportado.sql

Al ejecutar este comando se generará un archivo denominado ScottExportado.sql que incluirá todas las instrucciones SQL necesarias para recrear tanto la base de datos como sus datos.

Vamos a ver el contenido del archivo ScottExportado.sql:

Además, es factible exportar solamente una selección de tablas mediante la opción --tables. Por ejemplo, si solo se desea exportar la tabla dept, se podría emplear el comando siguiente:

mysqldump -u scott -p --tables scott dept > ScottExportado2.sql

Al ejecutar el comando anterior se generará un archivo llamado ScottExportado2.sql que incluirá solamente las instrucciones SQL requeridas para recrear la tabla dept y sus datos correspondientes.

Importación de datos con mysql

Vamos a crear un usuario nuevo y una base de datos en MySQL. El usuario se llamará scottimportado y tendrá los permisos necesarios para importar datos a la base de datos scott.

create database scottimportado;
create user scottimportado;
grant all privileges on *.* to 'scottimportado'@'localhost' identified by 'passwd' with grant option;
flush privileges;

A fin de importar el archivo ScottExportado.sql que generamos previamente, podemos ejecutar el comando siguiente:

mysql -u scottimportado -p scottimportado < ScottExportado.sql

Vamos a comprobar que se han importado correctamente los datos:

select * from dept;

5. Intenta realizar operaciones similares de importación y exportación con las herramientas proporcionadas con Postgres desde línea de comandos, documentando el proceso.

En PostgreSQL, existe una herramienta denominada pg_dump que facilita la exportación de datos desde la línea de comandos.

Exportación de datos con pg_dump

La herramienta pg_dump se emplea para generar copias de seguridad de bases de datos PostgreSQL. Con ella, es posible exportar la totalidad de la base de datos o solo algunos objetos específicos, como tablas, vistas o funciones. Para llevar a cabo una exportación completa de una base de datos, es posible usar el siguiente comando:

pg_dump -U postgres -d scott > /var/lib/postgres/ejer5/ScottExportado.sql

Este comando creará un fichero llamado ScottExportado.sql que contiene toda la estructura y los datos de la base de datos.

Vamos a ver el contenido del fichero ScottExportado.sql:

Observamos que incluso se exportaron las tablas creadas para fines de auditoría.

Otra opción disponible es exportar únicamente un conjunto específico de tablas mediante la opción --table. Por ejemplo, si deseas exportar solo la tabla dept, puedes utilizar el siguiente comando:

pg_dump -U postgres --table scott.dept > /var/lib/postgres/ejer5/ScottExportado2.sql

En el archivo ScottExportado2.sql solo se incluirá la estructura y los datos correspondientes a la tabla dept.

Importación de datos con PostgreSQL

Crearemos una nueva base de datos llamada scottimportado.

create database scottimportado with owner postgres;
grant all privileges on database scottimportado to postgres;

Una vez finalizado el proceso de exportación, se procede a la importación del archivo ScottExportado.sql recién creado, para lo cual se puede ejecutar el siguiente comando:

psql -U postgres -d scottimportado -f /var/lib/postgres/ejer5/ScottExportado.sql

Para verificar que la importación de datos se ha realizado correctamente, vamos a realizar una consulta:

select * from dept;
select * from emp;
select * from salgrade;

6. Exporta los documentos de una colección de MongoDB que cumplan una determinada condición e impórtalos en otra base de datos.

Si necesitas exportar documentos de una colección en MongoDB que cumplan con ciertos criterios, puedes utilizar el comando mongoexport junto con algunas opciones adicionales.

Imaginemos que disponemos de una colección llamada prueba1 en la base de datos Prueba, y deseamos exportar todos aquellos documentos que presenten un valor de salario superior a 3000. Para llevar a cabo esta tarea, podemos emplear el siguiente comando:

Esta base de datos es la que he usado:

use Prueba

db.prueba1.insertMany( [

{ nombre: "Roberto", sal: 2000 },

Comando mongoexport:

mongoexport -u olucas -p root --db Prueba --collection prueba1 --query '{ "sal": { "$gt": 3000 } }' --out exportacion.json --authenticationDatabase admin

Mediante la ejecución de este comando, se generará una copia de seguridad en el directorio actual de la terminal. El nombre del archivo resultante se establecerá en función de la fecha y hora de la exportación, y los datos se almacenarán en formato JSON.

Si necesitas importar los datos previamente exportados en otra base de datos, puedes recurrir al uso del comando mongoimport. Por ejemplo, si deseas importar los datos en una nueva base de datos denominada Prueba2, puedes ejecutar el siguiente comando:

Antes crearé la base de datos Prueba2:

7. SQLLoader es una herramienta que sirve para cargar grandes volúmenes de datos en una instancia de ORACLE. Exportad los datos de una base de datos completa desde Postgres a texto plano con delimitadores y emplead SQLLoader para realizar el proceso de carga de dichos datos a una instancia ORACLE. Debéis documentar todo el proceso, explicando los distintos ficheros de configuración y de log que tiene SQL*Loader.

Si se necesita exportar los datos de una base de datos completa de Postgres en un archivo de texto plano con delimitadores, es posible utilizar el siguiente comando (en este ejemplo he elegido las tablas dept y salgrade del esquema scott):

COPY dept TO '/var/lib/postgresql/scott/dept.csv' DELIMITERS',';
COPY salgrade TO '/var/lib/postgresql/scott/salgrade.csv' DELIMITERS',';

Una vez obtenido el archivo de texto plano que contiene los datos, es posible emplear SQL*Loader para cargarlos en una instancia de Oracle.

SQL*Loader es una herramienta que se utiliza mediante la línea de comandos, y que requiere de archivos de configuración para especificar los detalles de la carga de datos. A continuación se detalla el proceso completo para configurar y utilizar SQL*Loader con el fin de cargar los datos exportados desde Postgres en una instancia de Oracle, empleando el esquema scott.

Para poder llevar a cabo la carga de datos desde Postgres hacia Oracle mediante SQL*Loader, resulta necesario crear previamente en Oracle aquellas tablas que se correspondan con las estructuras de las tablas de Postgres. Es crucial asegurarse de que los nombres de las columnas sean idénticos y que los tipos de datos sean compatibles.

create table dept (
        deptno number(2) constraint pk_dept PRIMARY KEY,
        dname varchar2(14),
        loc varchar2(13)
);

create table salgrade (
        grade number,
        losal number,
        hisal number
);

Se deberá crear un archivo de control específico para cada tabla que se desee cargar mediante SQL*Loader y guardarlos en la carpeta /home/oracle/control.

nano dept.ctl

LOAD DATA
INFILE '/home/oracle/scott/dept.csv'
INTO TABLE dept
FIELDS TERMINATED BY ','
TRAILING NULLCOLS
(deptno, dname, loc)

nano salgrade.ctl

LOAD DATA
INFILE '/home/oracle/scott/salgrade.csv'
INTO TABLE salgrade
FIELDS TERMINATED BY ','
TRAILING NULLCOLS
(grade, losal, hisal)

Me pasaré los ficheros por scp creados en Postgres a Oracle y los meteré en la ruta que he puesto en los ficheros de control.

La ejecución de SQL*Loader se realizará mediante la línea de comandos, en la cual se indicarán los archivos de control y parámetros correspondientes.

sqlldr scott2/tiger control=/home/oracle/control/dept.ctl log=/home/oracle/log/dept.log
sqlldr scott2/tiger control=/home/oracle/control/salgrade.ctl log=/home/oracle/log/salgrade.log

Prueba de la tabla "dept".

Tras ejecutar el comando correspondiente, se mostrará en pantalla un mensaje informativo.

Es posible observar en la carpeta de logs la creación de un archivo que contiene información detallada sobre la carga de datos realizada mediante SQL*Loader.

Prueba de la tabla "salgrade".

Acto seguido accederé a la base de datos y comprobaré que se han cargado los datos correctamente.

La herramienta SQL*Loader es muy útil porque nos permite cargar datos de manera eficiente y simple. Podemos cargar datos desde archivos de texto y trabajar con datos de diferentes orígenes. Además, podemos cargar datos en tablas que contienen claves foráneas y relaciones, lo que nos brinda mayor flexibilidad. En resumen, esta herramienta es una excelente opción para aquellos que necesitan cargar grandes cantidades de datos en sus bases de datos de manera eficiente y rápida.
